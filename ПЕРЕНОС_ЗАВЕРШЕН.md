## 🔍 ДЕТАЛЬНЫЙ АНАЛИЗ JNI КОДА

### 1. Структура Main Loop (jni/jni/main.cpp)

Проанализирована полная цепочка выполнения:

```cpp
// 1. Получение PlayerManager через getInstance
playerManager = helper.getInstance(libUnity.start + 151547120, true, 0x0);

// 2. Получение списка игроков и локального игрока
playersList = mem.read<uint64_t>(playerManager + 0x28);
localPlayer = mem.read<uint64_t>(playerManager + 0x70);

// 3. Получение ViewMatrix (4-шаговая цепочка)
ptr1 = mem.read<uint64_t>(localPlayer + 0xE0);
ptr2 = mem.read<uint64_t>(ptr1 + 0x20);
ptr3 = mem.read<uint64_t>(ptr2 + 0x10);
viewMatrix = mem.read<Matrix>(ptr3 + 0x100);

// 4. Получение локальной позиции (3-шаговая цепочка)
pos_ptr1 = mem.read<uint64_t>(localPlayer + 0x98);
pos_ptr2 = mem.read<uint64_t>(pos_ptr1 + 0xB8);  // ← КЛЮЧЕВОЙ ОФФСЕТ!
localPosition = mem.read<Vector3>(pos_ptr2 + 0x14);  // ← КЛЮЧЕВОЙ ОФФСЕТ!

// 5. Итерация по игрокам (List-based)
playerCount = mem.read<int>(playersList + 0x20);
playerPtr1 = mem.read<uint64_t>(playersList + 0x18);

for (int i = 0; i < playerCount; i++) {
    player = mem.read<uint64_t>(playerPtr1 + 0x30 + 0x18 * i);

    // 6. Получение позиции игрока (3-шаговая цепочка)
    pos_ptr1 = mem.read<uint64_t>(player + 0x98);
    pos_ptr2 = mem.read<uint64_t>(pos_ptr1 + 0xB8);  // ← КЛЮЧЕВОЙ ОФФСЕТ!
    position = mem.read<Vector3>(pos_ptr2 + 0x14);  // ← КЛЮЧЕВОЙ ОФФСЕТ!

    // 7. Отрисовка ESP
    functions.esp.Update(position, health, team, ...);
}
```

// 1. Get PlayerManager

playerPtr1 = mem.read<uint64_t>(playersList + 0x18);
